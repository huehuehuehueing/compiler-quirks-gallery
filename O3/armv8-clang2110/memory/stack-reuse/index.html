    stack-reuse¶

    Details

    Source: memory/stack-reuse.c Compiler: armv8-clang2110 (ARM) Scenario: O3 - Aggressive Optimization (-O3)

    Source Code¶

    stack-reuse.c
    /*
 * Stack slot reuse: non-overlapping locals share stack space.
 *
 * The compiler analyzes variable lifetimes and reuses
 * stack slots for variables that are never live at the same time.
 */
#include <stdio.h>

void example(int flag)
{
    if (flag) {
        /* This buffer is only live in this branch */
        char buffer_a[100];
        snprintf(buffer_a, sizeof(buffer_a), "Path A: %d", flag);
        printf("%s\n", buffer_a);
    } else {
        /* This buffer may reuse the same stack slot as buffer_a
         * since they're never both alive at the same time.
         */
        char buffer_b[100];
        snprintf(buffer_b, sizeof(buffer_b), "Path B: %d", flag);
        printf("%s\n", buffer_b);
    }

    /* Total stack usage may be ~100 bytes, not 200 */
}

    Explanation¶

    the key insight here is stack slot reuse - the compiler allocates only one 100-byte buffer on the stack instead of two, because the two buffers never exist at the same time.

    Stack Frame Setup¶

    sub     sp, sp, #128
stp     x29, x30, [sp, #112]
add     x29, sp, #112

    The function allocates 128 bytes on the stack (the sp register points to the top of the stack, and subtracting grows it downward). It then saves the frame pointer (x29) and return address (x30) at offset 112, establishing a stack frame. The frame pointer is updated to point to this saved location, creating a reference point for the function.

    Branch Selection (if/else)¶

    mov     w3, w0              // Save the flag parameter
cbz     w0, .LBB0_2         // If flag == 0, jump to else branch
adrp    x2, .L.str          // Load address of "Path A: %d"
add     x2, x2, :lo12:.L.str
b       .LBB0_3             // Jump to common code
.LBB0_2:
adrp    x2, .L.str.2        // Load address of "Path B: %d"
add     x2, x2, :lo12:.L.str.2
.LBB0_3:

    The cbz instruction (Compare and Branch if Zero) checks if w0 (the flag parameter) is zero. If it is, execution jumps to .LBB0_2 (the else branch). Otherwise, it falls through to load the "Path A" string address. Both branches converge at .LBB0_3 with the appropriate format string in x2.

    The Critical Optimization: Single Buffer Reuse¶

    add     x0, sp, #12         // Buffer address = sp + 12
mov     w1, #100            // Buffer size = 100
bl      snprintf            // Call snprintf
add     x0, sp, #12         // Same buffer address
bl      puts                // Call puts

    This is the key pattern: both buffer_a and buffer_b use the same stack location (sp + 12). The compiler analyzed that: - buffer_a is only used in the if-branch - buffer_b is only used in the else-branch - They're never both alive simultaneously

    So instead of allocating 200 bytes total (100 + 100), it allocates just 128 bytes and reuses the same 100-byte slot. The buffer sits at offset 12 from the stack pointer (leaving 12 bytes for other data, likely alignment padding).

    Stack Frame Cleanup¶

    ldp     x29, x30, [sp, #112]
add     sp, sp, #128
ret

    The saved frame pointer and return address are restored, the stack is deallocated (128 bytes), and the function returns.

    Why This Matters¶

    Without this optimization, the function would need 200+ bytes of stack space. With it, only 128 bytes are needed. The compiler's lifetime analysis (enabled by -O3 optimization) recognizes that the two buffers have non-overlapping lifetimes and safely reuses the stack slot. This is a form of register/stack allocation optimization - the compiler minimizes memory usage by understanding when variables are no longer needed.
