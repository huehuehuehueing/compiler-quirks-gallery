    static-memcpy¶

    Details

    Source: string-literals/static-memcpy.c Compiler: vc_v19_44_VS17_14_x64 (x86/x64 (Windows)) Scenario: O3 - Aggressive Optimization (-O3)

    Source Code¶

    static-memcpy.c
    #include <string.h>
#include <stdio.h>

void foo(void)
{
    char buf[5];

    memcpy(buf, "AAAA", 4);
    buf[sizeof(buf) - 1] = '\0';

    /* make sure the above is not treated as dead code */
    puts(buf);
}

    Explanation¶

    the key insight is that this code is heavily instrumented with security checks, which is why it looks more complex than you might expect for such a simple function.

    Stack Setup and Security Cookie¶

    The function starts by allocating 72 bytes of stack space: 

    sub     rsp, 72

    This large allocation (much more than the 5-byte buffer needs) is because MSVC is using stack buffer overflow protection. The compiler stores a security cookie—a random value that acts as a guard—to detect if a buffer overflow corrupts the stack:

    mov     rax, QWORD PTR __security_cookie    ; Load the security cookie
xor     rax, rsp                             ; Mix it with stack pointer
mov     QWORD PTR __$ArrayPad$[rsp], rax    ; Store it on the stack

    This cookie will be verified before the function returns to catch any overflow attempts.

    The memcpy Call¶

    The function sets up three parameters for memcpy using the x64 calling convention (parameters go in rcx, rdx, r8):

    mov     r8d, 4                               ; Third param: size = 4
lea     rdx, OFFSET FLAT:$SG10443           ; Second param: source = "AAAA"
lea     rcx, QWORD PTR buf$[rsp]            ; First param: destination buffer
call    memcpy

    The lea (Load Effective Address) instruction calculates addresses without actually accessing memory—it just computes where things are located.

    Range Check Before Array Access¶

    Here's where the security instrumentation becomes visible. Before writing to buf[4], the compiler inserts a bounds check:

    mov     eax, 1                               ; Start with 1
imul    rax, rax, 4                          ; Multiply by 4 (result: 4)
mov     QWORD PTR $T1[rsp], rax              ; Store 4 in temporary
cmp     QWORD PTR $T1[rsp], 5                ; Compare: is 4 < 5?
jae     SHORT $LN3@foo                       ; If not, jump to error
jmp     SHORT $LN4@foo                       ; Otherwise skip error

    This is checking that the array index (4, which is sizeof(buf) - 1) is within bounds (less than 5). If the check fails, it calls __report_rangecheckfailure. This is runtime bounds checking—the compiler is verifying array accesses at runtime to catch out-of-bounds writes.

    Null Terminator and Function Call¶

    Once the bounds check passes, the code writes the null terminator:

    mov     rax, QWORD PTR $T1[rsp]              ; Load the index (4)
mov     BYTE PTR buf$[rsp+rax], 0            ; Write 0 to buf[4]
lea     rcx, QWORD PTR buf$[rsp]             ; Load buffer address
call    puts                                  ; Print the string

    Security Cookie Verification¶

    Before returning, the function verifies the security cookie wasn't corrupted:

    mov     rcx, QWORD PTR __$ArrayPad$[rsp]    ; Load stored cookie
xor     rcx, rsp                             ; Mix with current stack pointer
call    __security_check_cookie              ; Verify it matches

    If the cookie was overwritten by a buffer overflow, this check will fail and terminate the program.

    Key Observations¶

    • This is -O3 optimized code, but the security instrumentation dominates the assembly output
    • The seemingly inefficient operations (like mov eax, 1; imul rax, rax, 4 to compute 4) are the compiler's way of generating the bounds check
    • MSVC's /GS (buffer security check) flag adds runtime protection that makes the code larger and slower, but catches buffer overflows before they can exploit the program
    • The 72-byte stack allocation includes space for the buffer (5 bytes), alignment padding, the security cookie, and temporary variables for the bounds check
