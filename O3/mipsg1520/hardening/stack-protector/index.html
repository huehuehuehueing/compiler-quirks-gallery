    stack-protector¶

    Details

    Source: hardening/stack-protector.c Compiler: mipsg1520 (MIPS) Scenario: O3 - Aggressive Optimization (-O3)

    Source Code¶

    stack-protector.c
    /* @gallery-hints
 *   extra-flags: -fstack-protector-all
 *   compiler-exclude: vc_v19_44_VS17_14_x64, vc_v19_44_VS17_14_x86
 */

/*
 * Stack Protector (Stack Canary)
 *
 * With -fstack-protector-all the compiler inserts a canary value
 * between local variables and the saved return address. Before the
 * function returns, the canary is checked; if it was overwritten
 * (e.g. by a buffer overflow), the program aborts.
 *
 * Look for: canary load from %fs:0x28 (or %gs:0x14 on 32-bit),
 * comparison before the function epilogue, and a call to
 * __stack_chk_fail on mismatch.
 */
#include <string.h>

void copy_input(const char *input)
{
    char buf[64];
    strcpy(buf, input);   /* intentionally unsafe for demonstration */
    (void)buf;
}

    Explanation¶

    the key insight here is that the compiler is implementing stack protection — a security mechanism that detects buffer overflows before they can corrupt the program.

    The Core Security Mechanism¶

    The function uses a canary value (a special number stored on the stack) to detect if a buffer overflow has occurred. Think of it like a tripwire: if a buffer overflow overwrites memory, it will likely overwrite the canary. Before returning, the function checks if the canary is still intact.

    Stack Frame Setup¶

    addiu   $sp,$sp,-104        # Allocate 104 bytes on stack
sw      $16,96($sp)         # Save $16 (callee-saved register)
sw      $31,100($sp)        # Save $31 (return address)

    The function allocates 104 bytes of stack space. In MIPS, $sp is the stack pointer register. The addiu instruction subtracts 104 from it (negative value = growing the stack downward). The function then saves two registers: $16 (which will hold the canary guard address) and $31 (the return address).

    Loading the Canary Guard¶

    lui     $16,%hi(__stack_chk_guard)    # Load high 16 bits of guard address
lw      $2,%lo(__stack_chk_guard)($16) # Load the actual canary value

    These instructions load the canary value from a special memory location (__stack_chk_guard). The lui instruction loads the upper portion of an address, and lw (load word) retrieves the actual value. The canary is stored in $2.

    Preparing for strcpy¶

    move    $5,$4               # Copy input pointer to $5 (2nd parameter)
addiu   $4,$sp,28           # Set $4 to buffer address (1st parameter)
sw      $2,92($sp)          # Store canary at offset 92 on stack

    The function sets up parameters for strcpy: $4 gets the buffer address (at stack offset 28), and $5 gets the input pointer. Crucially, the canary value is stored at offset 92 — positioned between the 64-byte buffer and the saved return address, so any overflow will hit it first.

    The strcpy Call¶

    jal     strcpy              # Jump and link to strcpy function
nop                         # Delay slot (no operation)

    jal calls strcpy, which copies the input string into the buffer. If the input is too long, it will overflow the buffer and overwrite the canary.

    Canary Verification (The Security Check)¶

    lw      $3,92($sp)          # Load stored canary from stack
lw      $2,%lo(__stack_chk_guard)($16) # Load fresh canary value
bne     $3,$2,$L5           # Branch if not equal (mismatch detected)

    Before returning, the function loads the canary it stored earlier and compares it with a fresh copy from __stack_chk_guard. If they don't match (bne = branch if not equal), execution jumps to label $L5.

    Normal Return Path¶

    lw      $31,100($sp)        # Restore return address
lw      $16,96($sp)         # Restore $16
jr      $31                 # Jump to return address
addiu   $sp,$sp,104         # Deallocate stack (delay slot)

    If the canary is intact, the function restores the saved registers and returns normally. The addiu instruction executes in the delay slot (a MIPS quirk where the instruction after jr executes before the jump).

    Failure Path¶

    $L5:
jal     __stack_chk_fail    # Call abort function
nop

    If the canary was corrupted, the program calls __stack_chk_fail, which terminates the program to prevent the overflow from causing further damage.

    Key Observations¶

    • Security overhead: The canary mechanism adds instructions for loading, storing, and checking the guard value. This is the cost of protection against buffer overflows.
    • Stack layout: The 104-byte allocation accommodates the 64-byte buffer plus space for the canary and saved registers.
