    fast-math¶

    Details

    Source: arithmetic/fast-math.c Compiler: clang1910 (x86/x64) Scenario: O3 - Aggressive Optimization (-O3)

    Source Code¶

    fast-math.c
    /*
 * Fast-math optimizations: -ffast-math / -Ofast effects.
 *
 * These flags allow the compiler to reorder floating-point
 * operations, assuming associativity (which floats don't have).
 * Can change results or break special value handling.
 */

/* With -ffast-math, this may be reordered and give different results */
float sum_array(const float *arr, int n)
{
    float sum = 0.0f;
    for (int i = 0; i < n; i++)
        sum += arr[i];
    return sum;
}

/* NaN check may be optimized away with -ffast-math */
int is_nan(float x)
{
    /* With -ffast-math, compiler assumes no NaN values exist,
     * so this comparison may be optimized to always return 0.
     */
    return x != x;
}

/* Division by variable may become multiplication by reciprocal */
float divide_by_y(float x, float y)
{
    /* With -ffast-math: may become x * (1.0f / y)
     * Less accurate but faster on some architectures.
     */
    return x / y;
}

/* Algebraic simplification (inexact with floats!) */
float algebraic(float a, float b, float c)
{
    /* With -ffast-math: (a + b) + c may be reordered to a + (b + c)
     * These can give different results with floating point!
     */
    return (a + b) + c;
}

    Explanation¶

    the key insight is that this code demonstrates loop unrolling and remainder handling in the first function, while the other functions show straightforward floating-point operations.

    sum_array - The Main Optimization Pattern¶

    The most important thing to understand here is loop unrolling: instead of adding one array element per loop iteration, the compiler processes 8 elements per iteration, then handles any remaining elements separately.

    Initial setup: 

    test    esi, esi          // Check if n <= 0
jle     .LBB0_1           // If so, jump to return 0.0
    The esi register holds the parameter n (by calling convention, integer parameters go in rdi, rsi, rdx, etc.). If n is zero or negative, we skip to .LBB0_1 which returns 0.0 in xmm0.

    Preparing for the main loop: 

    mov     edx, esi          // edx = n
mov     eax, edx
and     eax, 7            // eax = n % 8 (remainder)
cmp     esi, 8
jae     .LBB0_8           // If n >= 8, use unrolled loop
    The compiler calculates how many elements won't fit into groups of 8 (n % 8). If we have at least 8 elements, it jumps to the unrolled loop at .LBB0_8.

    The unrolled loop body (.LBB0_9): 

    addss   xmm0, dword ptr [rdi + 4*rcx]      // Add arr[i]
addss   xmm0, dword ptr [rdi + 4*rcx + 4]  // Add arr[i+1]
addss   xmm0, dword ptr [rdi + 4*rcx + 8]  // Add arr[i+2]
...
addss   xmm0, dword ptr [rdi + 4*rcx + 28] // Add arr[i+7]
add     rcx, 8                              // Move to next group of 8
cmp     rdx, rcx
jne     .LBB0_9
    This processes 8 floats per iteration. The addss instruction adds a single-precision float (32-bit) to xmm0. The memory addresses use 4*rcx because each float is 4 bytes, so arr[i] is at offset 4*i from the base pointer rdi.

    Handling the remainder (.LBB0_6): 

    lea     rcx, [rdi + 4*rcx]  // Calculate address of first remaining element
xor     edx, edx            // edx = 0 (counter)
.LBB0_6:
addss   xmm0, dword ptr [rcx + 4*rdx]
inc     rdx                 // Increment counter
cmp     rax, rdx            // Compare with remainder count
jne     .LBB0_6
    After the unrolled loop, this processes the leftover elements (0-7 of them) one at a time.

    is_nan - NaN Detection¶

    xor     eax, eax           // Clear eax (set return value to 0)
ucomiss xmm0, xmm0         // Compare float with itself
setp    al                 // Set al = 1 if parity flag is set
ret

    The trick here: a NaN value is not equal to itself. The ucomiss instruction compares xmm0 (the float parameter) with itself. For normal numbers, this comparison is false, so the parity flag isn't set. For NaN, the comparison sets the parity flag. The setp instruction sets al to 1 if the parity flag is set, otherwise 0. This returns 1 for NaN, 0 otherwise.
