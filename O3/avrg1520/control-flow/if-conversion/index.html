    if-conversion¶

    Details

    Source: control-flow/if-conversion.c Compiler: avrg1520 (AVR) Scenario: O3 - Aggressive Optimization (-O3)

    Source Code¶

    if-conversion.c
    /*
 * If-Conversion (branch to conditional move)
 *
 * Short if-else chains that select between two values can be
 * replaced with conditional-move instructions (cmov on x86).
 * This eliminates a branch, avoiding potential misprediction
 * penalties on modern super-scalar CPUs.
 *
 * Look for: cmovl / cmovge / cmovne instead of jl / jge / jne
 * with a branch over a mov.
 */

int min(int a, int b)
{
    if (a < b)
        return a;
    return b;
}

int clamp(int x, int lo, int hi)
{
    if (x < lo)
        x = lo;
    if (x > hi)
        x = hi;
    return x;
}

int abs_val(int x)
{
    return x < 0 ? -x : x;
}

    Explanation¶

    the key insight is that AVR doesn't have conditional move instructions, so the compiler uses conditional branches instead. Let me walk through each function.

    Understanding AVR Registers and Calling Convention¶

    First, some context: AVR is an 8-bit architecture, so 16-bit integers (like int) require two 8-bit registers. By convention: - r24:r25 holds the first parameter and return value (low byte in r24, high byte in r25) - r22:r23 holds the second parameter - r20:r21 holds the third parameter - r18:r19 are general-purpose work registers

    min(int a, int b) Function¶

    cp r22,r24          // Compare b_low with a_low
cpc r23,r25         // Compare b_high with a_high (with carry)
brge .L2            // Branch if b >= a (skip the swap)
mov r24,r22         // Copy b_low to return register
mov r25,r23         // Copy b_high to return register
.L2:
ret

    What's happening: The function compares the two parameters and conditionally moves the smaller value into the return registers.

    • cp and cpc perform a comparison (subtraction without storing the result). The cpc instruction includes the carry flag from the previous comparison, allowing 16-bit comparison with two 8-bit instructions.
    • brge (branch if greater or equal) skips the move if b >= a
    • If the branch is not taken (meaning a < b), the two mov instructions copy b into the return registers
    • If the branch is taken, the return registers already contain a, so we skip the move and return

    clamp(int x, int lo, int hi) Function¶

    mov r18,r24         // Copy x to work register r18:r19
mov r19,r25
cp r18,r22          // Compare x with lo
cpc r19,r23
brge .L5            // Branch if x >= lo (skip clamping to lo)
mov r18,r22         // If x < lo, set x = lo
mov r19,r23
.L5:
mov r24,r18         // Copy x to return registers
mov r25,r19
cp r20,r24          // Compare hi with x
cpc r21,r25
brge .L6            // Branch if hi >= x (skip clamping to hi)
mov r24,r20         // If x > hi, set x = hi
mov r25,r21
.L6:
ret

    What's happening: This performs two sequential clamping operations.

    The compiler uses r18:r19 as a temporary holding area for x. It then: 1. Compares x with lo and conditionally replaces x with lo if needed 2. Moves the result to the return registers 3. Compares hi with the current value and conditionally replaces it with hi if needed 4. Returns

    Notice the pattern: each if-statement becomes a comparison, a conditional branch, and a conditional move. This is the standard approach on AVR since there are no conditional move instructions.

    abs_val(int x) Function¶

    sbrs r25,7          // Skip next instruction if bit 7 of r25 is set
ret                 // Return immediately if positive
neg r25             // Negate high byte
neg r24             // Negate low byte
sbc r25,__zero_reg__ // Subtract carry from high byte (complete 16-bit negation)
ret

    What's happening: This uses a clever optimization for absolute value.

    • sbrs (skip if bit set) checks the sign bit (bit 7 of the high byte). If the number is positive, bit 7 is 0, so we skip the negation and return immediately.
    • If the number is negative, bit 7 is 1, so we execute the negation:
    • neg r24 negates the low byte
    • neg r25 negates the high byte
    • sbc r25,__zero_reg__ subtracts the carry from the high byte to complete the two's complement
