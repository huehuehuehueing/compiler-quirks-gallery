    if-conversion¶

    Details

    Source: control-flow/if-conversion.c Compiler: armv7-clang2110 (ARM) Scenario: O3 - Aggressive Optimization (-O3)

    Source Code¶

    if-conversion.c
    /*
 * If-Conversion (branch to conditional move)
 *
 * Short if-else chains that select between two values can be
 * replaced with conditional-move instructions (cmov on x86).
 * This eliminates a branch, avoiding potential misprediction
 * penalties on modern super-scalar CPUs.
 *
 * Look for: cmovl / cmovge / cmovne instead of jl / jge / jne
 * with a branch over a mov.
 */

int min(int a, int b)
{
    if (a < b)
        return a;
    return b;
}

int clamp(int x, int lo, int hi)
{
    if (x < lo)
        x = lo;
    if (x > hi)
        x = hi;
    return x;
}

int abs_val(int x)
{
    return x < 0 ? -x : x;
}

    Explanation¶

    the key insight here is that the compiler has eliminated branches entirely by using conditional execution - a powerful ARM32 feature that lets instructions execute only when certain conditions are true.

    The Core Pattern: Conditional Execution Instead of Branches¶

    In ARM32, most instructions can be executed conditionally based on the CPU's condition flags (set by comparison instructions like cmp). This is fundamentally different from x86, where you typically need explicit branch instructions. The compiler is using this to avoid jumps, which can cause pipeline stalls if the CPU mispredicts which branch will be taken.

    Function-by-Function Breakdown¶

    min(int, int) - Simplest Case¶

    cmp     r0, r1          // Compare: is r0 < r1?
movge   r0, r1          // If r0 >= r1, move r1 into r0
bx      lr              // Return

    What's happening: - r0 and r1 hold the two input parameters (ARM32 calling convention passes first two arguments in these registers) - cmp r0, r1 compares them and sets the condition flags - movge r0, r1 means "move if greater-or-equal": if r0 >= r1, copy r1 into r0 - If the condition is false (meaning r0 < r1), the movge instruction is simply skipped, and r0 already contains the correct value - bx lr returns to the caller

    No branch needed - the conditional move handles both cases without jumping.

    clamp(int, int, int) - Two Conditional Moves¶

    cmp     r0, r1          // Is r0 < lo (r1)?
movle   r0, r1          // If r0 <= lo, set r0 = lo
cmp     r0, r2          // Is r0 > hi (r2)?
movge   r0, r2          // If r0 >= hi, set r0 = hi
bx      lr              // Return

    What's happening: - Parameters: r0 = x, r1 = lo, r2 = hi - First comparison clamps the lower bound: if x < lo, replace it with lo - Second comparison clamps the upper bound: if x > hi, replace it with hi - Again, no branches - just two conditional moves executed sequentially

    abs_val(int) - Conditional Negation¶

    cmp     r0, #0          // Is r0 < 0?
rsbmi   r0, r0, #0      // If negative (mi = minus), negate it
bx      lr              // Return

    What's happening: - cmp r0, #0 compares r0 against zero - rsbmi r0, r0, #0 is "reverse subtract if minus": this instruction computes 0 - r0 and stores it back in r0, but only if the negative flag is set (meaning r0 was negative) - If r0 is positive or zero, the rsbmi is skipped and r0 is returned unchanged

    Why This Matters (The Optimization Story)¶

    Without -O3 optimization, the compiler would generate code with actual branch instructions (bne, blt, etc.) that jump over mov instructions. This creates a problem on modern CPUs: if the branch predictor guesses wrong, the CPU has to flush its pipeline and restart, wasting cycles.

    By using conditional execution, the compiler eliminates branches entirely. The CPU executes both the comparison and the conditional move in sequence without any jumps - the condition flags from cmp simply control whether the next instruction takes effect. This is much faster on modern processors because there's no branch prediction penalty.

    This is ARM32's equivalent to x86's conditional move instructions (cmov), which the comment mentions - it's the same optimization strategy, just using ARM's conditional execution syntax.
