    auto-vectorize¶

    Details

    Source: simd/auto-vectorize.c Compiler: cg152 (x86/x64) Scenario: Ofast - Fast Math Optimization (-Ofast)

    Source Code¶

    auto-vectorize.c
    /*
 * Auto-vectorization: loops converted to SIMD operations.
 *
 * Compiler transforms scalar loops into vector operations
 * (SSE, AVX, NEON, etc.) processing multiple elements at once.
 */

/* Simple loop - prime candidate for vectorization */
void add_arrays(float *dst, const float *a, const float *b, int n)
{
    for (int i = 0; i < n; i++) {
        dst[i] = a[i] + b[i];
    }
    /* With -O3: becomes SIMD adds processing 4-8 floats at once */
}

/* Reduction - trickier to vectorize but often done */
float sum_array(const float *arr, int n)
{
    float sum = 0.0f;
    for (int i = 0; i < n; i++) {
        sum += arr[i];
    }
    return sum;
    /* May use horizontal add instructions or partial sums */
}

/* Loop with dependency - CANNOT be vectorized */
void fibonacci_array(int *arr, int n)
{
    arr[0] = 0;
    arr[1] = 1;
    for (int i = 2; i < n; i++) {
        /* Each iteration depends on previous two - serial */
        arr[i] = arr[i-1] + arr[i-2];
    }
}

/* Conditional in loop - may use masked operations */
void clamp_array(float *arr, int n, float min, float max)
{
    for (int i = 0; i < n; i++) {
        if (arr[i] < min) arr[i] = min;
        else if (arr[i] > max) arr[i] = max;
    }
}

    Explanation¶

    the key insight is that this code demonstrates auto-vectorization the compiler has transformed scalar loops into SIMD (Single Instruction Multiple Data) operations that process multiple floating-point values simultaneously.

    add_arrays: The Core Vectorization Pattern¶

    The most important pattern here is how the compiler handles the simple element-wise addition loop:

    movups  xmm0, XMMWORD PTR [rsi+rax]    // Load 4 floats from array 'a'
movups  xmm2, XMMWORD PTR [rdx+rax]    // Load 4 floats from array 'b'
addps   xmm0, xmm2                     // Add all 4 pairs at once
movups  XMMWORD PTR [rdi+rax], xmm0    // Store 4 results

    This is the vectorized loop (.L5). Instead of adding one float per iteration, the addps instruction (add packed single-precision) adds four 32-bit floats in parallel. The xmm0 and xmm2 registers are 128-bit SIMD registers that hold four floats each.

    Why this matters: With -Ofast, the compiler recognizes that array elements are independent each dst[i] depends only on a[i] and b[i], not on other iterations. This independence allows vectorization.

    Alignment Checks (The Setup Code)¶

    Before entering the vectorized loop, the compiler performs several checks:

    lea     rax, [rdi-4]           // Calculate address for alignment check
mov     r8, rax
sub     r8, rsi                // Check if 'a' and 'dst' are aligned
cmp     r8, 8
jbe     .L3                    // If not well-aligned, use scalar fallback

    These checks verify that the arrays are sufficiently aligned in memory. If they aren't, the compiler jumps to .L3, which uses a scalar fallback loop that processes one float at a time with addss (add scalar single-precision).

    Handling Remainder Elements¶

    After the vectorized loop processes elements in groups of 4, there may be 13 remaining elements. The code handles this with a remainder loop (.L4):

    mov     eax, ecx
and     eax, -4                // Round down to nearest multiple of 4
sub     ecx, eax               // Calculate how many elements remain

    The and eax, -4 instruction masks off the lower 2 bits, effectively computing n & ~3 to find the largest multiple of 4 that doesn't exceed n. Any leftover elements are processed one at a time.

    sum_array: Vectorized Reduction¶

    This function demonstrates a more complex optimization: horizontal reduction combining multiple SIMD results into a single scalar value.

    The vectorized loop accumulates four partial sums:

    movups  xmm2, XMMWORD PTR [rax]    // Load 4 floats
addps   xmm0, xmm2                 // Add to running sum (4 values)

    After the loop, xmm0 contains four partial sums that must be combined into one. The compiler uses a series of shuffle and add instructions:

    movhlps xmm1, xmm0                 // Move high 64 bits to low 64 bits
addps   xmm1, xmm0                 // Add: combines elements
shufps  xmm0, xmm1, 85             // Shuffle to align remaining elements
addps   xmm0, xmm1                 // Final add

    These instructions perform a horizontal reduction they add the four floats within the register together. The movhlps (move high to low packed single) and shufps (shuffle packed single) instructions rearrange elements so that addps can combine them progressively.

    Any remaining elements (when n % 4 != 0) are added individually with addss.
