    static-memcpy¶

    Details

    Source: string-literals/static-memcpy.c Compiler: mipsg1520 (MIPS) Scenario: Ofast - Fast Math Optimization (-Ofast)

    Source Code¶

    static-memcpy.c
    #include <string.h>
#include <stdio.h>

void foo(void)
{
    char buf[5];

    memcpy(buf, "AAAA", 4);
    buf[sizeof(buf) - 1] = '\0';

    /* make sure the above is not treated as dead code */
    puts(buf);
}

    Explanation¶

    the key insight here is that the compiler has eliminated the memcpy call entirely and directly embedded the string data into the stack, then calls puts. This is an aggressive optimization enabled by -Ofast.

    Stack Setup and String Initialization¶

    The function starts by allocating 40 bytes of stack space: 

    addiu   $sp,$sp,-40

    This creates room for the local buf array (5 bytes) plus space for the return address and other data.

    The compiler then constructs the string "AAAA\0" directly in memory without calling memcpy:

    li      $2,1094778880                 # 0x41410000
addiu   $2,$2,16705
sw      $2,24($sp)
sb      $0,28($sp)

    Here's what's happening: - li (load immediate) loads 0x41410000 into register $2. The hex value 0x41 is the ASCII code for 'A' - addiu adds 16705 (which is 0x4141 in hex) to create the full 4-byte value 0x41414141 — this represents "AAAA" in memory - sw (store word) writes this 4-byte value to the stack at offset 24 from $sp — this is where buf starts - sb (store byte) writes a zero byte ($0) at offset 28 — this is the null terminator at buf[4]

    Function Call Setup¶

    addiu   $4,$sp,24
jal     puts

    The compiler calculates the address of buf (stack pointer + 24) and places it in $4. By MIPS calling convention, $4 is the first argument register, so this passes the buffer pointer to puts. The jal instruction jumps to puts while saving the return address.

    Return and Stack Cleanup¶

    lw      $31,36($sp)
jr      $31
addiu   $sp,$sp,40

    The return address is restored from the stack into $31, then jr jumps back to the caller. The addiu deallocates the stack space (note it executes in the delay slot after jr, which is standard MIPS practice).

    Key Optimization Pattern¶

    The -Ofast flag enabled constant folding and dead code elimination: the compiler recognized that memcpy with a constant string and constant size can be replaced with direct memory writes. This eliminates an entire function call, making the code faster and smaller.
