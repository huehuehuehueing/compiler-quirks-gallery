    static-init-1¶

    Details

    Source: string-literals/static-init-1.c Compiler: mipsg1520 (MIPS) Scenario: O2 - Standard Optimization (-O2)

    Source Code¶

    static-init-1.c
    /*
 * Static string initialization with concatenation.
 *
 * Adjacent string literals are concatenated at compile time.
 * The initializer may become a single memcpy or inline stores.
 */
#include <string.h>
#include <stdio.h>

void foo(void)
{
    /* Adjacent strings concatenated at compile time: "AAAABBBB" */
    char buf[100] = { "AAAA" "BBBB" };

    /* Make observable to prevent dead code elimination */
    puts(buf);
}

    Explanation¶

    the key insight here is that the compiler is using a hybrid approach to initialize the buffer: it loads the pre-concatenated string data from memory and copies it into the stack-allocated buffer using memset.

    String Data Setup¶

    The compiler first loads the concatenated string "AAAABBBB\0" from the read-only data section ($LC0):

    lui     $3,%hi($LC0)           # Load upper 16 bits of string address into $3
lwl     $7,%lo($LC0)($3)       # Load left word (first 4 bytes: "AAAA")
addiu   $2,$3,%lo($LC0)        # Compute full address of string in $2
lwr     $7,3($2)               # Load right word (completes first 4 bytes)
lwl     $3,4($2)               # Load left word (next 4 bytes: "BBBB")
lwr     $3,7($2)               # Load right word (completes second 4 bytes)
lbu     $2,8($2)               # Load byte (null terminator)

    The lwl/lwr pair (Load Word Left/Right) is a MIPS technique for loading unaligned data. Together they load a full 32-bit word. The compiler loads the 8-byte string plus null terminator into registers $7, $3, and $2.

    Stack Frame Setup¶

    addiu   $sp,$sp,-136           # Allocate 136 bytes on stack
sw      $31,132($sp)           # Save return address (caller needs it to return)

    The function allocates 136 bytes of stack space and saves the return address $31 (which holds where to return after this function completes).

    Buffer Initialization¶

    li      $6,91                  # Load 91 (size to clear)
move    $5,$0                  # $5 = 0 (fill value)
addiu   $4,$sp,33              # $4 = address of buffer on stack
sw      $7,24($sp)             # Store first 4 bytes of string
sw      $3,28($sp)             # Store next 4 bytes of string
jal     memset                 # Call memset to zero-fill remaining space
sb      $2,32($sp)             # Store null terminator

    Rather than initializing the entire 100-byte buffer with the string, the compiler: 1. Stores the loaded string data directly to the stack 2. Calls memset to zero-fill the remaining 91 bytes (the rest of the 100-byte buffer)

    This is more efficient than copying the entire string byte-by-byte.

    Function Call and Return¶

    jal     puts                   # Call puts with buffer address
addiu   $4,$sp,24              # $4 = buffer address (delay slot)

lw      $31,132($sp)           # Restore return address
nop                            # Delay slot (no operation needed)
jr      $31                    # Jump to return address
addiu   $sp,$sp,136            # Deallocate stack (delay slot)

    The jal (Jump And Link) instruction calls puts, with the buffer address passed in $4 (the first argument register by MIPS calling convention). After puts returns, the function restores the return address and deallocates the stack before returning to the caller.

    Key pattern: Instructions in the delay slot (the instruction after jal and jr) execute before the jump takes effect—this is a MIPS architectural feature that allows the CPU to keep busy while preparing for the branch.
