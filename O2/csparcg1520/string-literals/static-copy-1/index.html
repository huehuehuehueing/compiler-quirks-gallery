    static-copy-1¶

    Details

    Source: string-literals/static-copy-1.c Compiler: csparcg1520 (SPARC) Scenario: O2 - Standard Optimization (-O2)

    Source Code¶

    static-copy-1.c
    /*
 * String copy with strcpy - compiler may inline as immediate stores.
 *
 * For small, known-length strings, strcpy becomes direct
 * memory stores with the string data embedded in instructions.
 */
#include <string.h>
#include <stdio.h>

void foo(void)
{
    char buf[100];
    strcpy(buf, "ABCDXXXXCCCC");

    /* Make observable to prevent dead code elimination */
    puts(buf);
}

    Explanation¶

    the key insight here is that the compiler has inlined the string copy operation - instead of calling strcpy, it directly embeds the string data into the function using immediate store instructions.

    Stack Frame Setup¶

    save    %sp, -200, %sp
add     %fp, -104, %o0

    The save instruction creates a new stack frame, allocating 200 bytes of local space. This is the SPARC calling convention - save simultaneously saves the caller's registers and sets up a new register window. The add instruction then calculates the address of the local buffer buf (at offset -104 from the frame pointer %fp) and places it in %o0, which is the first argument register by convention.

    String Data Encoding¶

    sethi   %hi(1094860800), %g2
sethi   %hi(1482184704), %g3
or      %g2, 836, %g2
or      %g3, 88, %g3
sethi   %hi(1128480768), %g1
or      %g1, 835, %g1

    This is where the compiler has embedded the string "ABCDXXXXCCCC" as immediate values. The sethi instruction loads the high 22 bits of a 32-bit value into a register, and or fills in the low 10 bits. These three pairs of instructions construct three 32-bit words containing the ASCII characters of the string: - %g2 and %g3 together hold 8 bytes - %g1 holds 4 more bytes

    This is an optimization - instead of calling strcpy at runtime, the compiler recognizes the string is a compile-time constant and directly encodes it.

    Storing the String Data¶

    std     %g2, [%fp-104]
st      %g1, [%fp-96]
stb     %g0, [%fp-92]

    The std instruction (store double) writes two 32-bit registers (%g2 and %g3) as an 8-byte block into the buffer. The st instruction stores the remaining 4 bytes from %g1. Finally, stb (store byte) writes a zero byte (%g0 always contains 0) to null-terminate the string.

    Function Call and Return¶

    call    puts, 0
 stb    %g0, [%fp-92]
return  %i7+8
 nop

    The call instruction invokes puts with the buffer address already in %o0. Note the stb instruction in the delay slot (the instruction after call that executes before the branch takes effect) - this is why the null terminator write appears here rather than before the call. The return instruction jumps back to the caller using %i7 (the saved return address), and nop fills the delay slot.

    Key optimization: This code demonstrates string constant inlining - the compiler avoided the overhead of a strcpy function call by directly embedding the string data as immediate values and storing them with efficient multi-byte store instructions.
