    fast-math¶

    Details

    Source: arithmetic/fast-math.c Compiler: armv8-clang2110 (ARM) Scenario: O2 - Standard Optimization (-O2)

    Source Code¶

    fast-math.c
    /*
 * Fast-math optimizations: -ffast-math / -Ofast effects.
 *
 * These flags allow the compiler to reorder floating-point
 * operations, assuming associativity (which floats don't have).
 * Can change results or break special value handling.
 */

/* With -ffast-math, this may be reordered and give different results */
float sum_array(const float *arr, int n)
{
    float sum = 0.0f;
    for (int i = 0; i < n; i++)
        sum += arr[i];
    return sum;
}

/* NaN check may be optimized away with -ffast-math */
int is_nan(float x)
{
    /* With -ffast-math, compiler assumes no NaN values exist,
     * so this comparison may be optimized to always return 0.
     */
    return x != x;
}

/* Division by variable may become multiplication by reciprocal */
float divide_by_y(float x, float y)
{
    /* With -ffast-math: may become x * (1.0f / y)
     * Less accurate but faster on some architectures.
     */
    return x / y;
}

/* Algebraic simplification (inexact with floats!) */
float algebraic(float a, float b, float c)
{
    /* With -ffast-math: (a + b) + c may be reordered to a + (b + c)
     * These can give different results with floating point!
     */
    return (a + b) + c;
}

    Explanation¶

    the key insight is that this code demonstrates loop unrolling and vectorization optimizations that process multiple array elements per iteration, plus straightforward implementations of the simpler functions.

    sum_array - The Main Optimization Story¶

    The most important pattern here is loop unrolling: instead of adding one float per iteration, the compiler processes 8 floats per loop cycle. This is a common -O2 optimization that reduces loop overhead.

    Initial setup and branching: 

    cmp     w1, #1          // Compare n with 1
b.lt    .LBB0_3         // If n < 1, jump to return 0
movi    d0, #0          // Initialize sum to 0.0
cmp     w1, #8          // Check if n >= 8
b.hs    .LBB0_4         // If yes, use fast vectorized path

    The function first checks if there are fewer than 8 elements. If so, it skips the optimized loop and goes to the scalar fallback.

    The vectorized loop (.LBB0_5): 

    ldp     q1, q2, [x10, #-16]    // Load 8 floats (2 SIMD registers, 4 floats each)
subs    x11, x11, #8           // Decrement counter by 8
add     x10, x10, #32          // Advance pointer by 32 bytes (8 floats)

    Here's what's happening: ldp (load pair) loads two 128-bit SIMD registers (q1 and q2), each containing 4 floats. That's 8 floats total per iteration.

    Extracting and summing individual floats: 

    mov     s3, v1.s[1]     // Extract float at index 1 from q1
fadd    s0, s0, s1      // Add float at index 0
mov     s4, v1.s[2]     // Extract float at index 2
fadd    s0, s0, s3      // Add the extracted float
// ... continues for all 8 floats

    The compiler extracts each of the 4 floats from q1, then each of the 4 from q2, and adds them sequentially to the running sum in s0. The loop repeats until all 8-element chunks are processed.

    Scalar cleanup (.LBB0_7 and .LBB0_8): 

    add     x10, x0, x9, lsl #2    // Calculate address of remaining elements
sub     x8, x8, x9             // Calculate how many elements remain
ldr     s1, [x10], #4          // Load one float
fadd    s0, s0, s1             // Add it
b.ne    .LBB0_8                // Loop if more remain

    If the array size isn't a multiple of 8, this loop handles the leftover 1-7 elements one at a time.

    is_nan - NaN Detection¶

    fcmp    s0, s0          // Compare float with itself
cset    w0, vs          // Set w0 to 1 if "overflow/NaN" flag is set
ret

    This is elegant: a NaN value is the only float that doesn't equal itself. The fcmp instruction sets the "overflow" condition flag (vs) when comparing NaN with anything (including itself). The cset instruction sets the return value to 1 if that flag is set, otherwise 0.

    divide_by_y - Direct Division¶

    fdiv    s0, s0, s1      // Divide s0 by s1
ret

    This is straightforward: fdiv performs floating-point division. With -O2, the compiler doesn't convert this to reciprocal multiplication (which would require -ffast-math). The division is computed directly.
