    static-copy-1¶

    Details

    Source: string-literals/static-copy-1.c Compiler: mipsg1520 (MIPS) Scenario: O0 - No Optimization (-O0)

    Source Code¶

    static-copy-1.c
    /*
 * String copy with strcpy - compiler may inline as immediate stores.
 *
 * For small, known-length strings, strcpy becomes direct
 * memory stores with the string data embedded in instructions.
 */
#include <string.h>
#include <stdio.h>

void foo(void)
{
    char buf[100];
    strcpy(buf, "ABCDXXXXCCCC");

    /* Make observable to prevent dead code elimination */
    puts(buf);
}

    Explanation¶

    the key insight here is that the compiler is manually copying the string data byte-by-byte into the stack buffer instead of calling strcpy. This is a common optimization where the compiler recognizes a constant string and inlines the copy operation.

    Stack Frame Setup¶

    The function starts by allocating space on the stack:

    addiu   $sp,$sp,-136        # Allocate 136 bytes on stack
sw      $31,132($sp)        # Save return address
sw      $fp,128($sp)        # Save frame pointer
move    $fp,$sp             # Set up new frame pointer

    The $sp register holds the stack pointer (the address of the top of the stack). By subtracting 136, the function reserves 136 bytes of local storage. The $31 register (return address) and $fp register (frame pointer) are saved because this function will call another function (puts), and these registers need to be restored when returning.

    Loading the String Data¶

    The next section loads the constant string "ABCDXXXXCCCC" from memory:

    lui     $2,%hi($LC0)        # Load upper 16 bits of string address into $2
lw      $5,%lo($LC0)($2)    # Load first 4 bytes of string into $5
addiu   $3,$2,%lo($LC0)     # Calculate address of string
lw      $4,4($3)            # Load next 4 bytes into $4
addiu   $3,$2,%lo($LC0)     # Calculate address again
lw      $3,8($3)            # Load next 4 bytes into $3

    The lui (Load Upper Immediate) instruction loads the high part of the string's address. Then three lw (Load Word) instructions fetch the string in 4-byte chunks. This is unoptimized code—notice the redundant address calculations (addiu $3,$2,%lo($LC0) appears twice).

    Copying to the Buffer¶

    The loaded data is then stored into the local buffer on the stack:

    sw      $5,24($fp)          # Store first 4 bytes at buf[0-3]
sw      $4,28($fp)          # Store next 4 bytes at buf[4-7]
sw      $3,32($fp)          # Store next 4 bytes at buf[8-11]

    The buffer buf[100] is located at offset 24 from the frame pointer. The string is 13 bytes long (including the null terminator), so the final byte is handled separately:

    addiu   $2,$2,%lo($LC0)     # Calculate string address
lbu     $2,12($2)           # Load byte 12 (the null terminator) as unsigned
nop                         # Delay slot (required after load)
sb      $2,36($fp)          # Store null terminator at buf[12]

    The lbu (Load Byte Unsigned) loads a single byte, and sb (Store Byte) stores it. The nop (no operation) is a delay slot—MIPS requires one instruction after a load before using the loaded value.

    Function Call Setup and Return¶

    addiu   $2,$fp,24           # Calculate address of buf (at offset 24)
move    $4,$2               # Move buf address to $4 (first argument register)
jal     puts                # Jump and link to puts function
nop                         # Delay slot

    By convention, the first argument to a function is passed in register $4. The jal (Jump And Link) instruction calls puts, and the nop in the delay slot is executed before the jump takes effect.

    Finally, the stack frame is cleaned up and the function returns:

    move    $sp,$fp             # Restore stack pointer
lw      $31,132($sp)        # Restore return address
lw      $fp,128($sp)        # Restore frame pointer
addiu   $sp,$sp,136         # Deallocate stack space
jr      $31                 # Jump to return address
nop                         # Delay slot
