    tail-call¶

    Details

    Source: control-flow/tail-call.c Compiler: armv8-clang2110 (ARM) Scenario: Os - Size Optimization (-Os)

    Source Code¶

    tail-call.c
    /*
 * Tail call optimization: recursive calls become jumps.
 *
 * When a function's last action is calling another function
 * (or itself), the call can become a jump, saving stack space.
 */

/* Tail-recursive factorial - can be optimized to a loop */
int factorial_tail(int n, int acc)
{
    if (n <= 1)
        return acc;
    /* Last action is the recursive call - tail position */
    return factorial_tail(n - 1, n * acc);
}

int factorial(int n)
{
    return factorial_tail(n, 1);
}

/* NOT tail recursive - multiplication after recursive call */
int factorial_not_tail(int n)
{
    if (n <= 1)
        return 1;
    /* Multiplication happens AFTER the call returns */
    return n * factorial_not_tail(n - 1);
}

/* Mutual tail recursion */
int is_even(unsigned n);
int is_odd(unsigned n);

int is_even(unsigned n)
{
    if (n == 0) return 1;
    return is_odd(n - 1);  /* Tail call */
}

int is_odd(unsigned n)
{
    if (n == 0) return 0;
    return is_even(n - 1);  /* Tail call */
}

    Explanation¶

    the key insight is that the compiler has eliminated the recursive function calls entirely and replaced them with loops. This is tail call optimization in action.

    The Core Pattern: Tail Calls Become Loops¶

    When a function's last action is calling another function (or itself), the compiler can replace that call with a jump back to the beginning of the loop. This saves stack space and avoids the overhead of function calls. With -Os (optimize for size), the compiler aggressively applies this optimization.

    Let me walk through the most important examples:

    is_even() and is_odd() - Mutual Tail Recursion Eliminated¶

    These two functions call each other recursively, but look at the assembly:

    is_even(unsigned int):
        cbz     w0, .LBB3_3          // if n == 0, jump to return 1
.LBB3_1:
        cmp     w0, #1               // compare n with 1
        b.eq    .LBB3_4              // if n == 1, jump to return 0
        sub     w0, w0, #2           // n = n - 2
        cbnz    w0, .LBB3_1          // if n != 0, loop back
.LBB3_3:
        mov     w0, #1               // return 1
        ret
.LBB3_4:
        mov     w0, wzr              // return 0 (wzr is zero register)
        ret

    What happened to the recursion? It's gone. Instead of calling is_odd(n-1), the code just subtracts 2 from n and loops back. The mutual recursion between is_even and is_odd has been completely eliminatedthe compiler recognized that alternating between them is equivalent to decrementing by 2 each time.

    Here's the flow: - cbz w0, .LBB3_3 "Compare and Branch if Zero": if n is 0, return 1 (even) - cmp w0, #1 followed by b.eq .LBB3_4 if n is 1, return 0 (odd) - sub w0, w0, #2 subtract 2 from n - cbnz w0, .LBB3_1 "Compare and Branch if Non-Zero": loop back if n is not zero

    This is a simple loop that decrements by 2 until reaching 0 or 1, then returns the appropriate value.

    factorial_tail() - Vectorized Loop¶

    The tail-recursive factorial is more complex because the compiler has applied vectorization (processing multiple values simultaneously using SIMD instructions):

    factorial_tail(int, int):
        subs    w8, w0, #2           // w8 = n - 2, set flags
        b.ge    .LBB0_2              // if n >= 2, jump to loop
        mov     w0, w1               // else return accumulator
        ret
.LBB0_2:
        // ... setup code ...
        movi    v0.4s, #1            // v0 = [1, 1, 1, 1] (4 32-bit values)
        dup     v2.4s, w0            // v2 = [n, n, n, n]
        // ... more setup ...
.LBB0_3:
        mul     v0.4s, v0.4s, v2.4s  // multiply 4 values at once
        sub     w8, w8, #4           // decrement counter by 4
        add     v2.4s, v2.4s, v3.4s  // adjust v2 for next iteration
        // ... loop condition check ...
        b.ne    .LBB0_3              // loop back
